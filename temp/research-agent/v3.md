# 我唯一会增加的一个核心对象

如果只能增加**一个对象**，我不会增加新的 Agent，也不会增加新的 Graph，更不会增加复杂的 Ontology，而是增加一个贯穿全流程的 **Research Session（研究会话）**。

它就是一次调查任务的工作上下文，例如：

```typescript
interface ResearchSession {
  goal: ResearchGoal;
  plan: ResearchPlan;

  identities: CanonicalIdentity[];

  evidence: Evidence[];

  graph: EvidenceGraph;

  findings: Finding[];

  gaps: Gap[];

  contradictions: Contradiction[];

  confidence: Confidence;

  report: ResearchReport;
}
```

这样整个 Proposal 的所有概念都会自然串联起来：

* **Research Plan** 驱动调查；
* **Evidence** 不断加入；
* **Identity Resolution** 更新统一身份；
* **Evidence Graph** 建立关联；
* **Gap** 和 **Contradiction** 在分析阶段产生；
* **Confidence** 评估结论可靠性；
* 最终生成 **Research Report**。

这个 `ResearchSession` 只是一个逻辑模型，不要求引入数据库、状态机或复杂框架，但它能够成为整个架构的主线，让 Proposal 从“一系列能力介绍”变成“一个完整、连贯的研究系统”。我认为这是在不增加复杂度的前提下，提升整体架构完整性最值得做的一处改进。
